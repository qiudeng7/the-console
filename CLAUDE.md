# AI 协作指南

## 工作风格

- **循序渐进，不要一次性抛出大量结果** - 逐步探索、逐步解释，让我跟上思路
- **先说明方案，再执行** - 技术选型时先解释不同方案的优缺点，等我确认后再实现

## 技术偏好

- **路径别名**：使用 `@/` 而非 `#/`
- **目录命名**：`pages/` 而非 `views/`
- **状态管理**：优先使用 Pinia，而非 composables
- **样式**：使用 Tailwind CSS

## 调试建议

遇到错误时，如果我已经分析过可能原因（如 "解构赋值导致 this 丢失"），优先按我的思路排查。

## 代码风格

- 保持简单实用，避免过度工程化
- 类型安全很重要，运行 `pnpm type-check` 验证
